## Deploy to GitHub Pages (Free Hosting)

### Step 1 — Create GitHub Account
Go to github.com and create a free account if you don't have one.

### Step 2 — Create New Repository
- Click the green "New" button
- Name it exactly: `solvia-website`
- Set to Public
- Click "Create repository"

### Step 3 — Upload Files
- Click "uploading an existing file"
- Drag ALL files from this folder into the upload area
- Click "Commit changes"

### Step 4 — Enable GitHub Pages
- Go to Settings → Pages
- Under "Source" select "Deploy from a branch"
- Select "main" branch, "/ (root)" folder
- Click Save

### Step 5 — Your site will be live at:
`https://[yourusername].github.io/solvia-website`

### Step 6 — Connect www.solviahealth.org
In your domain registrar (where you bought solviahealth.org):
Add these DNS records:
- Type A: 185.199.108.153
- Type A: 185.199.109.153
- Type A: 185.199.110.153
- Type A: 185.199.111.153
- Type CNAME: www → [yourusername].github.io

In GitHub Pages settings, add custom domain: www.solviahealth.org
Check "Enforce HTTPS"

That's it — www.solviahealth.org will be live within 24 hours!
